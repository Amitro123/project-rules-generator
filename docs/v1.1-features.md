# PRG v1.1 Features Guide

**Release Date:** February 2026  
**Version:** 1.1.0

## What's New

PRG v1.1 introduces three powerful new features to improve documentation quality and project planning:

1. **Quality Check** - AI-powered documentation scoring
2. **Roadmap Planning** - Generate project roadmaps from README
3. **Plan Status Tracking** - Visual progress tracking for plans

## Feature 1: Quality Check \ud83c\udd95

### Overview

Automatically analyze generated `.clinerules` files for quality and get actionable improvement suggestions.

### Quick Start

```bash
# Basic quality check
prg analyze . --quality-check

# With auto-fix for low-scoring files
prg analyze . --quality-check --auto-fix

# With verbose output for detailed breakdowns
prg analyze . --quality-check --verbose
```

### Scoring Criteria

Each file is scored across 5 criteria (20 points each, 100 total):

| Criterion | Description | Examples |
|-----------|-------------|----------|
| **Structure** | Headers, logical flow, no empty sections | H1/H2/H3 hierarchy, TOC |
| **Clarity** | Precise language, no fluff | Technical terms defined, clear examples |
| **Project Grounding** | References actual files/tools/commands | `src/main.py`, `pytest`, specific paths |
| **Actionability** | "How to" instructions | Step-by-step guides, executable commands |
| **Consistency** | Terminology matches project | Same naming conventions throughout |

### Score Interpretation

- **90-100**: \u2705 Excellent - Production-ready
- **85-89**: \u2705 Good - Minor improvements possible
- **70-84**: \u26a0\ufe0f Needs improvement - Several issues to address
- **0-69**: \u274c Poor quality - Major revisions needed

### Auto-Fix

Files scoring below 85 automatically get improvement patches when using `--auto-fix`:

```bash
prg analyze . --quality-check --auto-fix
```

**What it does:**
- Generates improved version using AI
- Applies fixes automatically
- Commits changes if git is available
- Shows before/after scores

## Feature 2: Roadmap Planning \ud83c\udd95

### Overview

Generate comprehensive project roadmaps from your README.md, organizing features into logical phases with tasks and subtasks.

### Quick Start

```bash
# Generate roadmap from README
prg plan --from-readme README.md

# Custom output location
prg plan --from-readme README.md --output docs/ROADMAP.md

# Use specific AI provider
prg plan --from-readme README.md --provider groq
```

### How It Works

1. **Extracts Features** - Parses README for Features, TODO, Roadmap sections
2. **Organizes Phases** - Groups related features into 3-5 logical phases
3. **Creates Tasks** - Breaks down each feature into actionable tasks
4. **Adds Subtasks** - Provides 2-4 subtasks for complex items

### Example Output

```markdown

---

  - [ ] Write implementation

  - [ ] Implement ETL logic

Ensure quality and document the project

- [ ] Write comprehensive tests
  - [ ] Unit tests
  - [ ] Integration tests
  - [ ] E2E tests

  - [ ] API docs
  - [ ] User guide
  - [ ] Examples
```

### Use Cases

- **Project Planning**: Convert README features into actionable roadmap
- **Team Alignment**: Share long-term vision with stakeholders
- **Progress Tracking**: Use with `--status` to track completion

## Feature 3: Plan Status Tracking \ud83c\udd95

### Overview

Track progress on existing plan files with visual progress bars, current task highlighting, and blocking task detection.

### Quick Start

```bash
# Show status of all plans in current directory
prg plan --status
```

### What It Shows

1. **Overall Progress**: Total tasks completed vs remaining
2. **Current Task**: Next task to work on
3. **Blocking Tasks**: Tasks with all subtasks done but not marked complete
4. **Phase Breakdown**: Progress per phase with visual bars

### Supported Plan Formats

Automatically detects and parses:
- `PLAN*.md`
- `PROJECT-ROADMAP.md`
- `ROADMAP.md`
- `*-PLAN.md`

### Blocking Task Detection

A task is "blocking" when:
- Task is not marked complete (`[ ]`)
- All subtasks are complete
- Indicates task needs manual review/completion

## Configuration

### Quality Check Settings

Create `src/config.py` to customize:

```python
from src.config import AnalyzerConfig

config = AnalyzerConfig(
    low_score_threshold=80,  # Default: 85
    ai_temperature=0.2,      # Default: 0.3
    max_suggestions=10       # Default: 5
)

analyzer = ContentAnalyzer(config=config)
```

## Troubleshooting

### Quality Check Issues

**Problem**: All scores are 50/100  
**Solution**: AI unavailable, using heuristic fallback. Set `GEMINI_API_KEY` for better results.

**Problem**: Auto-fix not working  
**Solution**: Ensure `--quality-check` is also enabled: `prg analyze . --quality-check --auto-fix`

### Roadmap Planning Issues

**Problem**: Empty roadmap generated  
**Solution**: Ensure README has a Features or TODO section with bullet points.

**Problem**: Too many/few phases  
**Solution**: Adjust `PlannerConfig.min_phases` and `max_phases`.

### Plan Status Issues

**Problem**: No plans found  
**Solution**: Create a plan first with `prg plan <task>` or `prg plan --from-readme`.

**Problem**: Progress not updating  
**Solution**: Manually edit plan file and mark tasks as `[x]` when complete.

## API Reference

### ContentAnalyzer

```python
from src.ai.content_analyzer import ContentAnalyzer

analyzer = ContentAnalyzer(provider='gemini', api_key='...')
report = analyzer.analyze('rules.md', content, project_path=Path('.'))

print(f"Score: {report.score}/100")
print(f"Status: {report.status}")
print(f"Suggestions: {report.suggestions}")

if report.patch:
    analyzer.apply_fix(Path('rules.md'), report.patch)
```

### ProjectPlanner

```python
from src.planning import ProjectPlanner

planner = ProjectPlanner(provider='gemini')
plan = planner.generate_roadmap_from_readme(
    Path('README.md'),
    project_path=Path('.')
)

plan.save(Path('PROJECT-ROADMAP.md'))
```

### PlanParser

```python
from src.planning import PlanParser

parser = PlanParser()
status = parser.parse_plan(Path('PLAN.md'))

print(f"Progress: {status.progress_percent}%")
print(f"Current: {status.current_task.description}")
print(f"Blocking: {len(status.blocking_tasks)}")
```
