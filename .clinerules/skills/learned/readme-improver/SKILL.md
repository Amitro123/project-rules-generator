# README Improver Skill

## Purpose
This skill provides the authoritative workflow for analyzing, structuring, and rewriting a project's `README.md` to ensure it meets professional open-source or enterprise standards.

## CRITICAL
> **WARNING: Non-Negotiable Constraints**
> 1. NEVER delete existing setup instructions or command examples without verifying they are obsolete.
> 2. ALWAYS maintain the original tone of the project unless explicitly asked to change it.
> 3. You MUST include a "Features", "Installation", and "Usage" section in your proposed structure.

## 1. Pre-Flight Checks
Before rewriting the README:
- Read the entire current `README.md` to understand the existing baseline.
- Scan the repository structure to identify key technologies, entry points, and recent changes that aren't documented.
- Review `references/readme_structure_guide.md` for the ideal template.

## 2. Drafting the Improvement
- Organize the document logically: Title & Badges -> Description -> Quick Start -> Features -> Installation -> Usage -> Contributing/License.
- Use clear, concise language. Remove fluff.
- Ensure all code blocks have the correct syntax highlighting tags (e.g., ````python````, ````bash````).
- Add or update badges if applicable (e.g., build status, license).

## 3. Applying and Verifying
- Present a summary of the structural changes proposed.
- Apply the changes using appropriate file editing tools.
- Run `scripts/validate_readme.py` to check for broken links or missing mandatory sections if available.
