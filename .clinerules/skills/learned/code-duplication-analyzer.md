# Skill: Code Duplication Analyzer

## Purpose
Improve code maintainability by identifying and reporting duplicated code in the project.

## Process

### 1. Scan Project Code
Run `pylint --output-format=parseable .` to scan the project code and generate a report on duplicated code.

### 2. Parse Report
Use `codeclimate --parse-report` to parse the report generated by pylint and extract relevant information.

### 3. Identify Duplicates
Use `clonehunter --identify-duplicates` to identify duplicated code based on the parsed report.

### 4. Report Findings
Generate a report on duplicated code, including file paths and lines of code.

## Output
- `duplicated_code_report.md` containing the report on duplicated code
- `duplicate_files.txt` containing a list of files with duplicated code

## Anti-Patterns
❌ Ignoring warnings from pylint → Run pylint with `--ignore-patterns` to configure warnings
❌ Not using a clone detection tool → Use clonehunter to identify duplicated code

## Tech Stack Notes
This skill requires the use of pylint, codeclimate, and clonehunter tools. Pylint is used to scan the project code and generate a report on duplicated code. Codeclimate is used to parse the report and extract relevant information. Clonehunter is used to identify duplicated code.
