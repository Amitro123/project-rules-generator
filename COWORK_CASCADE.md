   - tests/test_cache.py

**Steps:**
   1. Install redis[async] dependency
   2. Create RedisClient class with async methods
   3. Implement get/set/delete operations
   4. Add connection pooling

**Verify:** `pytest tests/test_cache.py --cov=80%`

**AC:** Cache hit rate > 80% in load test

**Risks:** Redis connection timeout, Memory overflow

### 2. Integrate cache with API endpoints (60min)

**Files:**
   - api/routes/users.py
   - api/routes/posts.py

**Depends:** #1

**Steps:**
   1. Add @cache_result decorator to endpoints
   2. Set appropriate TTL per endpoint
   3. Handle cache misses gracefully

**Verify:** `pytest tests/test_api_cache.py -v`

**AC:** Response time < 100ms for cached requests

**Risks:** Stale cache data

---

### 3. Add cache invalidation strategy (30min)

**Files:**
   - lib/cache_client.py
   - api/middleware.py

**Depends:** #1, #2

**Steps:**
   1. Implement invalidate_on_update decorator
   2. Add cache key pattern matching
   3. Create manual invalidation endpoint

**Verify:** `pytest tests/test_cache_invalidation.py`

**AC:** Cache updates within 1s of data change

**Risks:** Over-invalidation causing performance hit

---

## Tech Stack
- fastapi
- redis
- pytest
