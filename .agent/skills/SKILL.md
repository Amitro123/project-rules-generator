## CORE SKILLS

### analyze-code
Parse and analyze codebase for quality issues, patterns, and improvements.

**When to use:**
- Starting work on new codebase
- Refactoring before new feature
- Code review preparation

**Tools:** read, search, exec

**Usage:**
```

```

**Output:** Quality report with suggestions

### refactor-module
Refactor a specific module following project-rules-generator-rules.md guidelines.

**When to use:**

- Updating patterns

**Input:** Module path

**Output:**

### test-coverage
Run test suite and generate coverage analysis.

**When to use:**
- Before committing
- Periodically on main branch
- Before releases

**Tools:** exec (pytest, coverage)

**Usage:**
```bash
pytest --cov=src --cov-report=term-missing
```

**Output:** Coverage % and lines needing tests

**Output:** Updated documentation files
